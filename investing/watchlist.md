# Watchlist — hand-curated candidate ideas (NOT a buy list)

Names *you* choose to track for upcoming catalysts and Shariah compliance. This
file is **hand-curated — no script writes it.** Machine-discovered names live in
`leads.md` (auto-generated by discover.py, overwritten every run); promote a lead
here by copying its row, then write a setup card and screen it in Zoya/Musaffa.
The routine returns "ideas to investigate", never buy calls. You make every
decision; you verify every compliance status.

## How a name becomes a BUY-CANDIDATE (PM decision-logic pipeline)

Per docs/portfolio-manager-decision-logic.md, applied in order:
1. **Shariah knockout FIRST.** A business-activity knockout (conventional finance,
   alcohol, tobacco, gambling, defense primes, etc.) plus a rough 33/33 ratio
   pre-check. Names it flags are AVOID; missing data stays UNVERIFIED — unknown is
   NOT a pass. AAOIFI 30/30/5 and the <5% impermissible-revenue test are stricter
   than anything here: confirm each name in Zoya/Musaffa before any add.
2. **Edge = a completed setup card.** Not a `why` string. Until `setups/<ticker>.md`
   exists (entry trigger, stop logic, target logic, earnings plan, invalidation,
   status planned|live), an idea cannot exceed RESEARCH.
3. **Asymmetry gate.** Needs a defined target AND stop to compute reward:risk
   (>= reward_risk_min_swing).
4. **Catalyst-within-horizon gate.** A dated catalyst within catalyst_horizon_days.
5. **Shariah verified on the card.** `shariah.status: compliant`, screened within
   ~1 quarter. UNVERIFIED can never be BUY-CANDIDATE.
6. **Sizing.** Initial 1-2% at risk_per_trade_pct to the stop; ~20% hard cap; if
   earnings fall in the holding window, size for the gap (see the card's earnings_plan).
